Design Summary

1. Files & Structure

  • ../js/apply.js – 244 lines
  • ../js/visit.js – 56 lines
  • ../js/information.js – 54 lines

HTML, CSS, and JavaScript are separated into .html, ../css/, and ../js/ folders.

2. Event Handling (2 Techniques)

  • addEventListener (JavaScript)
    • apply.js line 22 – DOMContentLoaded init.
    • apply.js line 60 – click on division headers.
    • apply.js line 169 – click on rank buttons.
    • apply.js line 183 – submit link.
    • apply.js line 233 – clear link.
    • visit.js lines 3, 14, 53 – DOMContentLoaded, submit, reset.
    • information.js line 20 – DOMContentLoaded for text & video.
  • Inline HTML handler
    apply.html uses onclick="handleClearClick()" to call window.handleClearClick defined in apply.js line 175.

3. ES6 Features for Bonus

  • class
    apply.js line 4 – class ChosenGroup models one chosen internship (division, group, rank) as an object instead of loose variables.
  • map
    apply.js line 13 – const divisionNameMap = new Map([...]) maps form CSS class (e.g. forms-opt) to division names, used at line 127 to get the correct division.
  • set
    apply.js line 187 – const chosenRanks = new Set() stores unique ranks when submitting, then checks for duplicate ranks and missing ranks (gaps) before accepting the application.
  • let & const
    Used throughout to get block scope and prevent accidental reassignment:
    • apply.js lines 13, 25, 103 – configuration and DOM references.
    • visit.js lines 4–8, 29 – form elements and validated number.
    • information.js line 2 – promoMessages, line 9 – videoSources, lines 27 & 36 – let msgIndex, let vidIndex for rotation logic.

4. Page Logic (Short)

  • Apply: choose groups with ranks, prevent duplicates, detect gaps using Set, and show summary table.
  • Visit: validate date, time, and integer visitors, then call reserve() or show error.
  • Information: rotate promo text every 3 seconds and automatically switch between two videos.

5. CW2 Layout & Multi-column Design (CSS)

  • Common header / navigation / footer – shared styles are defined in ../css/common.css (214 lines). The gradient header and footer plus centered logo are in lines 16–44 and 103–132; the horizontal sticky navigation bar is in lines 48–74.
  • Home page two-column layout – the “Information” and “Map and Apply” blocks use a two-column flex layout in .info (home.css lines 17–33) together with the shared .multi-col rule (common.css lines 168–181).
  • About page welcome + text/table – the text container in the welcome block is limited to 80% of the page width and split into text + table via flexbox (.about-container in about.css lines 52–82). The hero image .about-image is responsive and never exceeds its container (about.css lines 74–81).
  • Visit page map + location – the map image and location text are arranged as a two-column block using .location-block (visit.css lines 52–70), while the visitor booking form is styled and aligned with the labels and inputs in lines 30–47.
  • Information page promo text + video – the promotional message and the video are placed side-by-side using a flex layout in .promo (information.css lines 6–24).
  • Consistent table styling & hover – all major tables share the same look: header row with background color and centered text via th, td and thead rules in about.css lines 93–118, information.css lines 26–38, apply.css lines 108–139, and contact.css lines 5–17. Table row hover effects are defined in about.css line 113, information.css lines 72–78, and apply.css lines 158–172.

6. Free Design Requirement 1 – Positioning Schemes

  • Sticky navigation bar – the main navigation uses position: sticky and top: 0 in common.css lines 48–54 so that the menu stays at the top when scrolling.
  • Relative container & absolute overlay – the About page full-width welcome image uses position: relative on .welcome (about.css line 65) to center the image across the viewport. The footer shine effect is implemented by footer::before with position: absolute in common.css lines 135–142.

7. Free Design Requirement 2 – CSS3 Techniques

  • Rounded corners & shadows (basic CSS3) – content blocks use border-radius and box-shadow, for example div, section in common.css lines 149–156, .about-text in about.css lines 68–75, and the chosen-group result table in apply.css lines 121–129.
  • Gradients & transforms – the header and footer use animated linear-gradient backgrounds in common.css lines 16–31 and 103–118. Navigation links apply transform: translateY(-2px) on hover (common.css lines 72–78) to give a lifting effect.
  • Transitions – smooth hover transitions are applied to navigation links (common.css line 66), division headings and forms on the Apply page (apply.css lines 42–44 and 95–101), and table rows in the result block (apply.css lines 158–172).
  • Animations – the header and footer gradients use keyframe animations headerGradient and footerGradient in common.css lines 23–31 and 109–117. The CW2 design page itself applies a simple fade-in animation to the whole body using @keyframes fadeIn and animation in design.css lines 17–24.

8. Free Design Requirement 3 – Stylesheet Organisation

  • Shared stylesheet – ../css/common.css (214 lines) holds all global styles: header, navigation bar, footer, base typography, and common multi-column layout rules.
  • Page-specific stylesheets – each CW2 page has its own CSS file imported or linked: home.css (50 lines), about.css (177 lines), apply.css (213 lines), visit.css (88 lines), information.css (83 lines), contact.css (37 lines), and design.css (24 lines). This keeps layout and visual styles modular and easier to maintain.
  • Print stylesheet – printApply.css (132 lines) is linked with media="print" to format only the header, footer, and result table when the Apply page is printed, hiding all unnecessary on-screen elements.
